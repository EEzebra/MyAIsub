# 战车智能体规则

## 编码规范

### 最小化修改原则

| 原则 | 说明 |
|------|------|
| 优先编辑 | 不创建新文件，除非必要 |
| 增量实现 | 小步快跑，逐步完善 |
| 保持原风格 | 与现有代码风格一致 |
| 避免重写 | 只修改需要改的部分 |

### 代码质量检查清单

- [ ] 语法正确，无 linter 错误
- [ ] 包含必要的 import/依赖
- [ ] 无硬编码敏感信息
- [ ] 无安全漏洞模式
- [ ] 代码可立即运行

## 架构审核流程

### 必须调用 architect 的场景

| 场景 | 风险等级 |
|------|----------|
| 目录结构调整 | 高风险 |
| 新增/删除模块 | 高风险 |
| 架构范式变更 | 高风险 |
| 跨文件重构 | 中风险 |
| 单文件结构调整 | 低风险 |
| 变量/函数重命名 | 低风险 |

### 审核调用模板

```markdown
## 架构审核请求

**发起 Agent**: coder
**变更类型**: [变更描述]
**影响范围**: [文件/目录列表]
**变更原因**: [为什么需要这个变更]

请 architect 审核。
```

### 审核结果处理

| 结果 | 行为 |
|------|------|
| 通过（低风险） | 记录变更日志，执行变更 |
| 需用户确认（高风险） | 等待用户批准 |
| 拒绝 | 不执行，说明原因 |

## 安全编码规范

### 禁止引入的模式

| 模式 | 替代方案 |
|------|----------|
| 命令注入 | 使用参数化调用 |
| XSS | 输出编码/转义 |
| eval() | 安全的解析器 |
| pickle 反序列化 | JSON/msgpack |
| 硬编码密钥 | 环境变量/配置文件 |
| SQL 拼接 | 参数化查询 |
| 路径遍历 | 路径规范化 |
| 不安全下载 | 校验来源和签名 |

### 安全检查 Hook 响应

```
Hook 拦截 → 查看拦截原因 → 修正代码 → 重新提交
```


## 错误处理规范

### Linter 错误处理

| 尝试次数 | 行为 |
|----------|------|
| 1 次 | 使用 read_lints 检查，修复错误 |
| 2 次 | 分析根本原因，换方法修复 |
| 3 次 | 上报 planner，请求支持 |

### 超过 3 次失败上报格式

```markdown
## 编码任务失败上报

**任务**: [任务描述]
**失败次数**: 3 次
**错误信息**:
1. [第一次错误]
2. [第二次错误]
3. [第三次错误]

**已尝试方案**: [描述已尝试的修复方法]

请 planner 协调解决。
```

## 触发检查清单

每次触发时按以下清单执行：
- [ ] 理解任务要求
- [ ] 检查是否需要架构审核
- [ ] 编写/修改代码
- [ ] 自检安全风险
- [ ] 使用 read_lints 检查错误
- [ ] 修复发现的问题（最多 3 次）
- [ ] 确认代码可运行
