# 测试智能体规则

## 测试决策流程

### 判断是否需要动态测试

```
任务确定性评估：
├── 明确、常规任务 → 自行执行静态 + 动态测试
└── 不确定、复杂任务 → 通知用户（愚者）确认测试范围
```

### 动态测试执行方式

| 方式 | 适用场景 |
|------|----------|
| 自行执行 | 任务明确、测试方法标准 |
| 提供方法，用户执行反馈 | 需要特定环境、用户有偏好 |

### 通知用户的时机

当以下情况发生时，应通知用户确认：

1. **测试范围不确定** - 需要澄清哪些测试是必要的
2. **测试方法有选择** - 多种可行方案，需用户决定
3. **测试风险较高** - 可能影响环境或数据
4. **用户明确要求** - 用户希望参与测试过程

## 测试策略层次

### Level 1: 静态分析（必做）
- 语法检查：`bash -n *.sh`
- 权限检查：`ls -la *.sh`
- 格式验证：YAML/JSON 结构

### Level 2: 动态执行测试（必做）
- **必须实际执行脚本**，而非仅读取内容
- 使用 `execute_command` 工具运行测试
- 模拟 stdin 输入进行功能验证

### Level 3: 端到端测试（推荐）
- 模拟完整用户场景
- 验证多组件协作
- 环境变量隔离测试

## Hook 测试规范

### 必须测试项

| 测试项 | 方法 | 示例 |
|--------|------|------|
| 脚本语法 | `bash -n script.sh` | 静态检查 |
| 执行权限 | `ls -la script.sh` | 权限验证 |
| **实际执行** | `echo '{...}' \| bash script.sh` | 动态测试 |
| 环境变量 | `VAR=value bash script.sh` | 隔离测试 |
| 路径解析 | 在非目标目录执行脚本 | 相对路径验证 |

### Hook 执行测试模板

```bash
# 测试 Hook 输出格式
echo '{"session_id":"test","transcript_path":"/tmp/test.jsonl"}' \
  | CODEBUDDY_PROJECT_DIR=/path/to/project \
  bash /path/to/hook/script.sh

# 验证输出
# - 检查退出码
# - 检查 stdout 格式
# - 检查 stderr 内容
```

## 常见测试遗漏

### 容易遗漏的测试点

1. **相对路径问题**
   - 脚本中的 `.claude/xxx` 等相对路径
   - 必须验证在非项目目录执行时的行为

2. **环境变量依赖**
   - `${CODEBUDDY_PROJECT_DIR}` 是否正确使用
   - 缺少环境变量时的 fallback 行为

3. **错误处理**
   - stdin 为空时的行为
   - 文件不存在时的行为
   - JSON 解析失败时的行为

4. **并发安全**
   - 多次执行是否会冲突
   - 临时文件命名是否唯一

## 置信度评估标准

| 覆盖程度 | 置信度 |
|----------|--------|
| 仅静态分析 | 60-70% |
| 静态 + 动态执行 | 80-90% |
| 静态 + 动态 + 端到端 | 95-100% |
