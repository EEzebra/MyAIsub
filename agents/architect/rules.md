# 皇帝智能体规则

## 架构定义与维护

### MyAI 架构定义

```
MyAI/
├── agents/         # 智能体定义（每个 agent 一个子目录）
├── commands/       # 命令集（按功能分类）
├── docs/           # 知识库（文档集中存储）
├── hooks/          # 事件脚本（条件触发）
├── rules/          # 规则（AI规则 + 项目规则）
├── skills/         # 技能（扩展能力）
├── spec/           # 场景流程（任务模板）
└── .claude/        # Claude/CodeBuddy 配置
```

### 架构一致性检查

| 检查项 | 标准 |
|--------|------|
| agent 目录结构 | 必须包含 AGENT.md 和 rules.md |
| 目录命名 | kebab-case 小写 |
| 文件命名 | kebab-case.md 或 大写.md (AGENT.md) |
| 规则优先级 | rules/ > spec/ > 临时 |

### 方案 B: 干净架构
**描述**: [方案描述]

### 方案 C: 务实平衡
**描述**: [方案描述]
**优势**: [优势列表]

### 方案设计原则

| 原则 | 说明 |
|------|------|
| 覆盖全谱 | 最小改动 → 干净架构 → 务实平衡 |
| 权衡透明 | 明确每种方案的优势/劣势/风险 |
| 有理有据 | 推荐方案需给出理由 |
| 用户决策 | 最终选择权在用户 |

## 风险评估标准

### 风险等级判定

| 等级 | 变更类型 | 特征 |
|------|----------|------|
| 低风险 | 命名变更、单文件调整 | 影响范围小，可逆 |
| 中风险 | 跨文件重构、模块内部调整 | 影响范围中等 |
| 高风险 | 目录结构调整、模块增删 | 影响范围大，难逆 |

### 风险处理流程

```
变更请求
    │
    ▼
评估风险等级
    │
    ├── 低风险 → 记录变更日志 → 放行
    │
    └── 高风险 → 暂停 → 用户确认
                      │
                      ├── 批准 → 记录变更日志 → 放行
                      └── 拒绝 → 撤销变更请求
```

## 变更日志规范

### arch-changelog.md 格式

```markdown
# 架构变更日志

| 日期 | 风险等级 | 变更描述 | 影响范围 | 决策者 |
|------|----------|----------|----------|--------|
| YYYY-MM-DD | 低/中/高 | [描述] | [文件/目录] | [用户/architect] |
```

### 必须记录的变更

- 所有目录结构调整
- 所有模块增删
- 所有命名规范变更
- 所有架构范式变更
- 高风险的任何变更

## 审核响应模板

### 低风险变更通过

```markdown
## 架构审核结果

**变更类型**: [变更描述]
**风险等级**: 低风险
**结果**: ✅ 通过

变更已记录到 docs/arch-changelog.md
```

### 高风险变更需确认

```markdown
## 架构审核结果

**变更类型**: [变更描述]
**风险等级**: 高风险
**影响范围**: [影响的文件/目录]
**结果**: ⏸️ 等待用户确认

**说明**: [为什么需要确认]
**建议**: [architect 的建议]

请确认是否执行此变更？
- [ ] 批准执行
- [ ] 拒绝执行
- [ ] 需要更多信息
```

### 变更拒绝

```markdown
## 架构审核结果

**变更类型**: [变更描述]
**结果**: ❌ 拒绝

**原因**: [拒绝原因]
**替代建议**: [替代方案]

请考虑替代方案后重新提交。
```

- [ ] 自己的目录结构必须完整（AGENT.md + rules.md）
