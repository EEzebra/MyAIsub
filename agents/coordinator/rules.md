# 教皇智能体规则

## 并行调度规范

### 并行判断标准

| 条件 | 决策 |
|------|------|
| 子任务间无依赖 | ✅ 可并行 |
| 子任务间有依赖 | ❌ 串行执行 |
| 部分有依赖 | 混合策略 |

### 并行数量限制

| 场景 | 建议并行数 |
|------|------------|
| 代码探索 | 2-4 个 subagent |
| 测试验证 | 2-3 个 subagent |
| 文档生成 | 2-3 个 subagent |
| 综合分析 | 3-4 个 subagent |

### 并行启动模板

```markdown
## 并行任务分配

**任务**: [主任务描述]
**并行数**: [N] 个 subagent

### Subagent 分配

| # | 名称 | 目标 | 工具 | 预期输出 |
|---|------|------|------|----------|
| 1 | [name] | [目标1] | [工具列表] | [输出格式] |
| 2 | [name] | [目标2] | [工具列表] | [输出格式] |
| N | [name] | [目标N] | [工具列表] | [输出格式] |

### 聚合规则
- 一致结果: [处理方式]
- 冲突结果: [解决策略]
```

## 结果聚合规范

### 聚合策略矩阵

| 结果类型 | 特征 | 策略 |
|----------|------|------|
| 一致结果 | 所有 subagent 结论相同 | 直接采用 |
| 互补结果 | 结论不冲突，互为补充 | 合并输出 |
| 冲突结果 | 结论矛盾 | 置信度优先，标注分歧 |
| 部分失败 | 部分 subagent 失败 | 使用成功结果，标记失败 |

### 冲突解决流程

```
发现冲突
    │
    ▼
评估各 subagent 置信度
    │
    ├── 置信度差异明显 → 采用高置信度结果
    │
    └── 置信度相近 → 标注分歧，请求人工判断
```

## 置信度评估规范

### 置信度计算因素

| 因素 | 权重 | 说明 |
|------|------|------|
| 证据完整性 | 40% | 是否有充分的代码/文档支撑 |
| 逻辑一致性 | 30% | 结论是否自洽 |
| 来源可靠性 | 20% | 信息来源是否可靠 |
| 验证程度 | 10% | 是否经过测试验证 |

### 置信度等级处理

| 置信度 | 等级 | 标记 | 处理 |
|--------|------|------|------|
| ≥ 90% | 高 | 🟢 绿色 | 直接采用 |
| 80-89% | 中 | 🟡 黄色 | 采用但标记待验证 |
| < 80% | 低 | 🔴 红色 | 需人工确认 |

### 置信度标注格式

```markdown
### 结论: [结论内容]
**置信度**: [XX%] [🟢/🟡/🔴]
**依据**: [支撑证据]
**待验证**: [需要验证的点]
```

## 与 planner 协作边界

### 职责划分

| 场景 | 主导 | 支持 | 说明 |
|------|------|------|------|
| 任务规划 | planner | - | 创建 task_plan.md |
| 任务拆解 | planner | coordinator | planner 拆解，coordinator 协助分析并行性 |
| 并行执行 | coordinator | - | 调度 subagent |
| 结果聚合 | coordinator | - | 合并输出 |
| 进度更新 | planner | coordinator | planner 更新状态 |

### 信息传递格式

**planner → coordinator**:
```markdown
## 并行任务请求

**来源**: planner
**任务**: [任务描述]
**子任务列表**:
1. [子任务1]
2. [子任务2]
3. [子任务3]

请 coordinator 执行并行调度。
```

**coordinator → planner**:
```markdown
## 并行执行完成

**任务**: [任务描述]
**状态**: 成功/部分失败/失败
**结果**: [聚合结论]

请 planner 更新 task_plan.md 状态。
```

## 错误处理规范

### Subagent 失败处理

| 失败数 | 处理 |
|--------|------|
| 1 个 | 记录失败原因，使用其他成功结果 |
| 多个但非全部 | 使用成功结果，标记失败项 |
| 全部失败 | 上报 planner，请求重新规划 |

### 上报格式

```markdown
## 并行任务失败上报

**任务**: [任务描述]
**失败 subagent 数**: [N]/[总数]

**失败原因**:
- [subagent1]: [失败原因]
- [subagent2]: [失败原因]

**建议**: [重新规划/换方法/请求支持]

请 planner 决策。
```

## 触发检查清单

每次触发时按以下清单执行：
- [ ] 分析任务可并行性
- [ ] 设计 subagent 分配
- [ ] 启动并行执行
- [ ] 收集各 subagent 结果
- [ ] 评估置信度
- [ ] 聚合结果
- [ ] 生成执行报告
- [ ] 通知 planner 更新状态
