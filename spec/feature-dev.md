# 场景：特性开发（Feature Dev）

## 触发条件
- 用户请求开发新功能或新特性
- 用户使用类似 "帮我开发..."、"实现一个..." 的表述
- 智能体判断为需要从零创建的编码任务

## 参与智能体
- planner（全程主导）、architect（架构设计）、coder（实现）、tester（验证）、maintainer（评分）

## 执行步骤

### Phase 1: 发现（Discovery）— planner
- **目标**：理解用户意图，明确功能需求
- **动作**：
  - 接收用户的功能描述
  - 分析功能边界和约束条件
  - 识别涉及的技术领域和所需技能
  - 记录到 findings.md
- **用户确认门**：无（自动进入下一阶段）

### Phase 2: 代码探索（Code Exploration）— planner + code-explorer(subagent)
- **目标**：了解现有代码库，发现相关文件和模式
- **动作**：
  - 并行探索：目录结构、相关文件、依赖关系
  - 识别可复用的代码和模式
  - 标记需要修改的文件
- **并行策略**：使用 CodeBuddy task 工具并行启动多个 code-explorer subagent
- **用户确认门**：无（自动进入下一阶段）

### Phase 3: 澄清问题（Clarification）— planner → 用户
- **目标**：消除需求中的歧义和不明确点
- **动作**：
  - 列出需要用户确认的问题清单
  - 提出 2~3 个关键问题
  - **必须等待用户回答后才能继续**
- **用户确认门**：**必须等用户回答**（此阶段暂停，直到用户回复）

### Phase 4: 架构设计（Architecture Design）— architect
- **目标**：设计方案，供用户选择
- **动作**：
  - 调用架构师智能体
  - 并行设计 3 个方案：
    - 方案 A：最小改动 — 快速交付，低风险
    - 方案 B：干净架构 — 可维护、可测试、可扩展
    - 方案 C：务实平衡 — 速度 + 质量的折中
  - 给出推荐方案和理由
- **用户确认门**：**必须等用户选择方案**（此阶段暂停，直到用户选择）

### Phase 5: 实现（Implementation）— coder
- **目标**：按选定方案编码实现
- **动作**：
  - 按计划逐步实现
  - 遵循规则和架构约束
  - 涉及架构修改时调用 architect 审核
  - 使用安全 hook 拦截风险代码
  - 增量实现，每个步骤可验证
- **用户确认门**：**必须在 Phase 4 用户批准后才开始**

### Phase 6: 质量审查（Quality Review）— tester
- **目标**：全面验证实现质量
- **动作**：
  - 并行执行多维度验证：
    - 功能正确性验证
    - Linter 错误检查
    - 文档一致性检查
    - 目录结构验证
  - 应用置信度过滤（>= 80% 才报告）
  - 低置信度问题启动二次确认
  - 问题反馈给 coder 修复 → 回归验证
- **并行策略**：多维度同时验证

### Phase 7: 总结（Summary）— planner + maintainer
- **目标**：交付成果，记录经验
- **动作**：
  - 更新 progress.md，标记任务完成
  - 更新 source.map（如有结构变化）
  - 触发 maintainer 进行会话评分
  - 总结本次开发的关键决策和经验教训
- **输出物**：
  - 功能代码
  - 测试报告
  - 架构变更日志（如有）
  - 会话评分报告

## 输出物清单
| 阶段 | 输出物 |
|------|--------|
| Phase 1 | findings.md（需求记录） |
| Phase 2 | findings.md（代码探索结果） |
| Phase 3 | 确认的需求文档 |
| Phase 4 | 架构方案文档（3 方案 + 推荐） |
| Phase 5 | 功能代码 |
| Phase 6 | 测试报告 |
| Phase 7 | progress.md + 评分报告 |

## 流程图
```
用户请求
  → Phase 1: 发现（自动）
  → Phase 2: 代码探索（自动，并行）
  → Phase 3: 澄清问题（★ 用户确认门）
  → Phase 4: 架构设计（★ 用户确认门）
  → Phase 5: 实现（★ 需 Phase 4 批准）
  → Phase 6: 质量审查（并行验证 + 置信度过滤）
  → Phase 7: 总结（评分 + 归档）
  → 完成
```
