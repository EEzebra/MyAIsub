# PROJECT.md — MyAI 项目入口配置

> MyAI 项目的核心规则与结构总览，供 AI 助手参考。

## 项目概述

MyAI 是一个 AI 辅助开发环境（v2.0），基于智能体协作、命令集、规则约束和技能扩展构建。

## 核心规则

### 行为规范
- 默认使用中文回复，简洁直接
- 不输出明文密码、API Key、Token 等敏感信息
- 不主动执行破坏性操作（`rm -rf`、`push --force` 等）
- 不自动 `git commit` / `git push`，需用户明确授权
- 增量修改，最小化改动，不重构大段代码
- 主动修复引入的 linter 错误（最多 3 轮）
- 3 次失败后停止，向用户说明情况

### 架构规范
- 所有架构变更必须经过 architect 智能体审核
- 编码智能体不得绕过架构审核流程
- 架构变更后必须更新 `docs/arch-changelog.md`

### 安全红线
- 禁止命令注入、XSS、eval/exec、pickle 反序列化、硬编码密钥、SQL 注入、路径遍历、不安全下载、危险 HTML（共 9 种模式）

### 规则优先级
全局规则 > 项目规则 > 场景规则 > 临时规则

## 智能体

| 智能体 | 代号 | 角色 | 核心职责 |
|--------|------|------|----------|
| planner | 魔术师 | 任务指挥中枢 | 任务理解、拆解规划、依赖分析、智能体调度、进度跟踪 |
| coder | 战车 | 代码执行引擎 | 代码编写、规则遵守、架构约束、技能调用、增量实现 |
| tester | 正义 | 质量守卫者 | 用例设计、功能验证、问题报告、回归验证、结果记录 |
| architect | 皇帝 | 核心守门人 | 架构定义、完整性监督、强制审核、分级风险处理 |
| maintainer | 星星 | 用户体验引擎 | 日志管理、Token 统计、偏好追踪、规则提取、会话评分 |

### 协作链路
```
用户任务 → planner(规划调度) → coder(编码) → architect(架构审核) → tester(验证) → maintainer(评分)
```

## 场景流程

根据任务类型匹配对应流程：

| 任务类型 | 流程文件 | 特殊机制 |
|----------|----------|----------|
| 特性开发 | `spec/feature-dev.md` | 7 阶段，3 个用户确认门 |
| 新项目初始化 | `spec/project-init.md` | 4 阶段，多方案架构设计 |
| Bug 修复 | `spec/bug-fix.md` | 4 阶段，并行验证 + 回归 |
| 技能开发 | `spec/skill-develop.md` | 5 阶段，skill-creator 辅助 |

## 目录职责

| 目录 | 职责 |
|------|------|
| `agents/` | 智能体定义（每个智能体一个子目录，含 AGENT.md） |
| `rules/` | 规则文件（AI 规则、项目规则） |
| `hooks/` | 条件触发的命令脚本（安全检查、架构管控等） |
| `commons/` | 通用命令脚本集（文件管理、项目运维等） |
| `skills/` | 技能归档（18 个扩展技能） |
| `spec/` | 场景流程定义（feature-dev/bug-fix/project-init/skill-develop） |
| `docs/` | 知识库（source.map、进度日志、变更日志等） |

## Hook 退出码协议

| 退出码 | 含义 | 行为 |
|--------|------|------|
| 0 | 放行 | 允许操作继续 |
| 1 | 警告 | 显示警告，不阻止操作 |
| 2 | 阻止 | 阻止操作，通知 AI 修正后重试 |
